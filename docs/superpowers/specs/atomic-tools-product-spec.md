# 原子工具集 — 产品需求文档 (PRD)

> 版本: v1.0 · 最后更新: 2026-04-16

---

## 1. 背景与动机

Translip 当前提供的是 **端到端 Pipeline 模式** —— 用户必须创建一个完整的任务才能使用系统能力。但实际使用中，用户经常只需要其中某一项独立能力：

- 音视频创作者只想**分离人声和伴奏**
- 会议记录只需要**语音转文字**
- 字幕翻译者只需要**文本翻译**
- 配音演员想试听**语音克隆效果**

这些需求无需走完整的多阶段流水线。**原子工具集 (Atomic Tools)** 将系统核心能力拆解为可独立使用的单功能工具，降低使用门槛、拓宽用户场景。

---

## 2. 目标用户

| 用户类型 | 典型场景 | 优先级 |
|---------|---------|--------|
| 音视频创作者 | 分离人声/伴奏，替换音轨 | P0 |
| 字幕工作者 | 语音转文字生成字幕，翻译字幕文件 | P0 |
| 配音/有声书制作者 | 语音克隆试听，文字转语音 | P0 |
| 开发/运维人员 | 检测媒体参数，诊断文件问题 | P1 |
| 播客/Podcast 制作者 | 音频混合，说话人分离 | P1 |

## 3. 工具清单 (方案A: 7个原子工具)

### 3.1 工具概览

| 编号 | 工具名称 | 类别 | 输入 | 产出 | 独立性 |
|-----|---------|------|------|------|--------|
| AT-1 | 人声/背景分离 | 🎵 音频处理 | 音频或视频文件 | voice.wav + background.wav | 完全独立 |
| AT-2 | 音频混合 | 🎵 音频处理 | 人声音频 + 背景音频 | 混合后的音频文件 | 完全独立 |
| AT-3 | 语音转文字 | 🗣️ 语音与文字 | 音频或视频文件 | 文字段落JSON + SRT字幕 | 完全独立 |
| AT-4 | 文本翻译 | 🗣️ 语音与文字 | 文本/SRT文件 | 翻译后的文本/SRT | 完全独立 |
| AT-5 | 语音合成 | 🗣️ 语音与文字 | 文本 + 可选参考音色 | 合成语音WAV | 完全独立 |
| AT-6 | 媒体信息探测 | 🎬 视频与媒体 | 任意音视频文件 | 媒体详情JSON | 完全独立 |
| AT-7 | 音视频合并 | 🎬 视频与媒体 | 视频文件 + 音频文件 | 合并后的MP4 | 完全独立 |

> 说话人识别作为**语音转文字 (AT-3)** 的内置选项（开关），不单独成为工具。

---

### 3.2 各工具详细规格

#### AT-1: 人声/背景分离 (Audio Separation)

**目标**: 从音视频文件中分离出人声轨和背景轨（音乐/环境声）。

**输入**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| file | File | ✅ | — | 上传的音频/视频文件 |
| mode | enum | ❌ | auto | auto / music / dialogue |
| quality | enum | ❌ | balanced | balanced / high |
| output_format | enum | ❌ | wav | wav / mp3 / flac |

**输出**:
- `voice.{format}` — 人声轨
- `background.{format}` — 背景轨
- 分离元信息（路由决策、使用的后端、耗时）

**交互流程**:
1. 用户上传文件（拖拽或点击）
2. 可选展开高级参数（模式、质量、格式）
3. 点击「开始分离」
4. 实时显示进度条 + 当前步骤
5. 完成后展示波形预览 + 在线试听 + 下载按钮

**容量限制**: 单文件最大 500MB，时长最长 60 分钟。

---

#### AT-2: 音频混合 (Audio Mixing)

**目标**: 将人声轨和背景轨按指定参数混合为最终音频。

**输入**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| voice_file | File | ✅ | — | 人声音频文件 |
| background_file | File | ✅ | — | 背景音频文件 |
| background_gain_db | number | ❌ | -8.0 | 背景音量增益（dB）|
| ducking_mode | enum | ❌ | static | static / sidechain |
| output_format | enum | ❌ | wav | wav / mp3 |

**输出**:
- `mixed.{format}` — 混合后的音频
- 混合参数报告

**交互流程**:
1. 分别上传人声和背景音频
2. 调节背景音量滑块（实时预览 dB 值）
3. 选择混音模式
4. 点击「开始混合」
5. 完成后在线试听 + 下载

**特色**: 可直接使用 AT-1 的输出（无需重复上传）— 工具间产物互通。

---

#### AT-3: 语音转文字 (Speech-to-Text)

**目标**: 对音视频文件进行语音识别，输出带时间戳的文字段落，可选开启说话人识别。

**输入**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| file | File | ✅ | — | 上传的音频/视频文件 |
| language | enum | ❌ | zh | 识别语言（zh/en/ja/auto等）|
| asr_model | enum | ❌ | small | tiny / base / small / medium / large-v3 |
| enable_diarization | bool | ❌ | false | 是否启用说话人识别 |
| generate_srt | bool | ❌ | true | 是否同时生成SRT字幕 |

**输出**:
- `segments.json` — 带时间戳的分段文字（开启说话人识别时含 speaker_label）
- `subtitles.srt` — SRT 格式字幕（可选）
- 转写统计（总时长、段落数、说话人数）

**交互流程**:
1. 用户上传文件
2. 选择语言（支持auto自动检测）
3. 可选开启「说话人识别」开关 → 展开提示文案："识别不同说话人并标注，适合多人对话/会议场景"
4. 选择模型精度（推荐small，大文件推荐tiny加速）
5. 点击「开始转写」
6. 实时显示进度条
7. 完成后展示：
   - 按时间线展示的文字段落（可折叠查看全文）
   - 说话人色彩标记（开启diarization时）
   - SRT文件下载、JSON下载、复制全文按钮

---

#### AT-4: 文本翻译 (Text Translation)

**目标**: 翻译用户输入的文本或上传的 SRT/文本文件。

**输入**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| text | string | ✅* | — | 直接输入文本（与 file 二选一）|
| file | File | ✅* | — | 上传 SRT 或纯文本文件（与 text 二选一）|
| source_lang | enum | ✅ | zh | 源语言 |
| target_lang | enum | ✅ | en | 目标语言 |
| backend | enum | ❌ | local-m2m100 | local-m2m100 / siliconflow |
| glossary | File | ❌ | — | 术语表文件（JSON）|

**输出**:
- 翻译结果文本
- 若输入为 SRT → 输出翻译后的 SRT 文件
- 若使用术语表 → 标注命中的术语对

**交互流程**:
1. 选择输入方式：直接输入文本 / 上传文件
2. 选择源语言 → 目标语言（语言对选择器，支持互换按钮 🔄）
3. 可选上传术语表
4. 可选切换翻译后端（本地 vs API）
5. 点击「翻译」
6. 结果区域左右对照展示原文和译文
7. SRT 模式下显示按段对照 + 下载按钮

**支持的语言**: zh ↔ en ↔ ja ↔ ko ↔ de ↔ fr ↔ es ↔ ru

---

#### AT-5: 语音合成 (Text-to-Speech)

**目标**: 将文本转化为语音，支持声音克隆。

**输入**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| text | string | ✅ | — | 要合成的文本内容 |
| language | enum | ❌ | auto | 合成语言（zh/en/ja/auto）|
| reference_audio | File | ❌ | — | 参考音色音频（10-30秒最佳）|

**输出**:
- `speech.wav` — 合成的语音文件
- 合成参数信息（时长、采样率）

**交互流程**:
1. 输入要合成的文本（多行文本框）
2. 选择目标语言
3. 可选上传参考音色 → 提示："上传一段10-30秒的清晰语音作为音色参考，系统将克隆该音色"
4. 点击「合成语音」
5. 完成后在线试听 + 下载
6. 若上传了参考音色，显示相似度评分

**特色**: 声音克隆是系统的核心亮点能力，在此工具中可直接体验。

---

#### AT-6: 媒体信息探测 (Media Probe)

**目标**: 检测音视频文件的格式参数。

**输入**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| file | File | ✅ | — | 上传的音视频文件 |

**输出**:
以结构化卡片展示：
- 📁 文件格式（MP4 / MKV / WAV / ...）
- ⏱️ 时长
- 🎥 视频流：编码、分辨率、帧率、码率
- 🔊 音频流：编码、采样率、声道数、码率
- 📐 容器信息

**交互流程**:
1. 用户拖入文件
2. 即时展示结果卡片（最轻量的工具，秒级响应）
3. 可复制为 JSON / 文本格式

---

#### AT-7: 音视频合并 (Video-Audio Muxing)

**目标**: 将音频轨替换或合并到视频文件中。

**输入**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| video_file | File | ✅ | — | 视频文件 |
| audio_file | File | ✅ | — | 音频文件 |
| video_codec | enum | ❌ | copy | copy（直通）/ libx264（重编码）|
| audio_codec | enum | ❌ | aac | aac |
| audio_bitrate | string | ❌ | 192k | 音频码率 |

**输出**:
- `output.mp4` — 合并后的视频文件
- 操作信息（输出时长、文件大小）

**交互流程**:
1. 上传视频文件和音频文件
2. 展开高级参数（编码选项）
3. 预览信息：显示视频时长 vs 音频时长的对比
4. 点击「合并」
5. 完成后提供下载

## 4. 工具间互通 (Cross-Tool Flow)

原子工具之间的产物可以零摩擦流转：

```
AT-1 (分离) ──→ voice.wav ──→ AT-3 (转文字)
              └→ background.wav ──→ AT-2 (混合) ←── 外部人声
AT-3 (转文字) ──→ segments.srt ──→ AT-4 (翻译)
AT-4 (翻译) ──→ 译文 ──→ AT-5 (语音合成) ──→ speech.wav ──→ AT-2 (混合)
AT-2 (混合) ──→ mixed.wav ──→ AT-7 (合并) ←── 原视频 ──→ final.mp4
```

**UI实现**:
- 每个工具的输出产物旁有「在其他工具中使用 →」快捷按钮
- 点击后跳转到目标工具页面，自动填入产物文件
- 产物在当前会话内保留，无需反复上传

## 6. 非功能需求

### 6.1 文件管理

- 上传文件临时存储在 `{TRANSLIP_CACHE_DIR}/atomic-tools/{session_id}/{tool_id}/input/`
- 产出文件存储在 `{TRANSLIP_CACHE_DIR}/atomic-tools/{session_id}/{tool_id}/output/`
- 会话文件自动清理策略：24小时后自动删除临时文件
- 单次上传文件限制：500MB

### 6.2 性能目标

| 工具 | 目标响应时间（10分钟音频） |
|------|------------------------|
| AT-6 媒体探测 | < 2s |
| AT-3 语音转文字 (tiny) | < 30s |
| AT-3 语音转文字 (small) | < 90s |
| AT-1 人声分离 (balanced) | < 120s |
| AT-4 文本翻译 (local) | < 10s |
| AT-5 语音合成 (30字) | < 15s |
| AT-7 音视频合并 | < 30s |
| AT-2 音频混合 | < 20s |

### 6.3 并发

- 单用户同一时刻最多同时运行 **2个** 原子工具任务
- 队列满时提示等待（不阻塞UI）

### 6.4 错误处理

- 所有工具统一错误展示样式
- 文件格式不支持 → 上传时即校验并提示
- 处理超时 → 提示"处理超时，建议降低质量或缩短音频"
- 模型未下载 → 提示并提供跳转到设置页面下载

## 7. 国际化

所有工具标题、描述、参数标签、提示文案、错误信息均需提供 **zh-CN** 和 **en-US** 翻译，遵循现有 i18n 架构。

## 8. 迭代计划

| 阶段 | 工具 | 优先级 |
|------|------|--------|
| Phase 1 | AT-6 媒体探测, AT-1 人声分离, AT-3 语音转文字 | P0 |
| Phase 2 | AT-4 文本翻译, AT-5 语音合成 | P0 |
| Phase 3 | AT-2 音频混合, AT-7 音视频合并 | P1 |
| Phase 4 | 工具间互通、会话产物管理 | P1 |

Phase 1 选择理由：AT-6 最轻量可作为技术验证；AT-1 和 AT-3 是最高频的独立需求。
